## ゴール

音(BGM・効果音)、デモ(OP/ED/中間)、グラフィック差し替え、セーブ/ロード
を実装し、「作ったゲームを最初から最後まで遊べて、続きから再開できる」状態
にする。

## 現状(想定: plan9 完了時点)

- 全コンテンツがエディターで制作可能。イベントの BGM変更/デモ起動は
  スタブ(発火ログのみ)。音は一切鳴らない。セーブ機能なし。

## スコープ

### やること

1. **音響基盤**: bevy_audio(Cargo feature 追加: "bevy_audio" と
   対応フォーマット "vorbis")。音量設定(BGM/SE/足音別、ミュート)。
2. **BGM**: フロアごとのBGM設定(オリジナルは17曲+なし)。
   CC0/CC-BY の音源からデフォルトBGM集を用意(**素材の記録ルール厳守**。
   Kevin MacLeod は CC-BY で帰属必要のため、CC0 優先なら
   OpenGameArt のミラー以外に itch/Kenney/FreePD 等から。ogg形式)。
   イベントの ChangeBgm を実体化。クロスフェード(1秒程度)。
3. **効果音**: 26種スロット(足音、ドア、攻撃ヒット、詠唱、光弾、落下、
   拾う、レベルアップ、しかけ床、ワープ等から優先実装)。アイテムの
   衝突時効果音、魔法の効果音の割当を実体化。
4. **デモシステム**: オープニング/エンディング(各1)+中間デモ(4)。
   内容: メッセージ行(参考: 最大160行)を1行ずつ表示するテキスト
   スクロール+背景色/背景画像(任意)+BGM指定。スキップ(Escape)。
   エンディング再生後はタイトルへ(タイトルは plan11 だが、暫定で
   「ENDと表示して入力待ち→メイン画面」でよい)。
   **デモエディター**: 行編集(複数行テキストエリア)+BGM選択+
   プレビュー再生。イベントの StartDemo を実体化。
5. **グラフィック差し替え機構**: プロジェクトディレクトリ配下の
   `assets_override/` に置いたファイルが組み込みアセットより優先される
   仕組み(壁/床/天井テクスチャ、アイテム・モンスター・パーティモデル、
   ポートレート静止画(置けば3Dバストより優先)、魔法シンボル画像、
   マウスカーソル相当は対象外)。エディターに差し替え状況の一覧画面。
6. **セーブ/ロード**: プレイ状態の保存(パーティ位置・向き・レベル、
   CharacterState、インベントリ、習得魔法、フラグ、ドア状態、
   モンスター状態、床のアイテム、経過サイクル、再生待ちタイマー)。
   スロット3以上、`saves/` にRONで保存。データ画面に「セーブ」を追加
   (オリジナルの「ディスク」)。起動時 `--load <slot>` と
   データ画面からのロード。
7. **システム環境設定**: ゲーム速度(サイクル秒の倍率 0.5/1.0/2.0)、
   音量、アイテム表示形式(簡易/詳細)、足音の有無。settings.ron に保存
   (プロジェクトではなくユーザー設定)。

### やらないこと(plan11へ)

- タイトル画面・プロジェクト選択・配布形式
- Windows ネイティブ実行スクリプト
- パフォーマンス最適化

## 設計メモ

- **セーブデータのバージョン**: `save_version` を持たせ、不一致は
  読み込み拒否+メッセージ(後方互換の努力は plan11 以降)。
- **決定論**: セーブ・ロードで挙動が変わらないよう、乱数は seeded RNG を
  リソース化して状態ごと保存する。
- **BGMアセットの容量**: ogg で合計 20MB 程度までに抑える(リポジトリ
  肥大防止。デフォルト曲は5〜8曲で開始し、17枠は設定上の枠として扱う)。
- 音は docker(PulseServer 経由)で鳴ることを確認する。鳴らない環境でも
  クラッシュしないこと。
- デモの背景画像は差し替え機構(assets_override)経由で供給。

## 実装ステップ

1. 音響基盤+効果音(主要10種程度をCC0で導入・記録)
2. BGM(フロア設定・エディターUI・ChangeBgm 実体化)
3. セーブ/ロード(ラウンドトリップのユニットテスト必須)
4. デモシステム+デモエディター+StartDemo 実体化
5. グラフィック差し替え機構+エディター一覧
6. システム環境設定
7. 検証シーン: `demo`(デモ再生中のテキスト)、`override`(差し替え
   テクスチャ適用状態)、セーブ→ロードの自動テスト
   (`DEEPGRID_DEBUG_SHOT=saveload`: セーブ→即ロード→位置一致を確認して
   スクショ)

## 受け入れ基準

1. ビルド完走、clippy/test 通過。
2. `DEEPGRID_DEBUG_SHOT=demo|override|saveload` 撮影+既存シーン全通過。
3. 手動確認: BGMがフロアで切り替わる(クロスフェード)、効果音、
   OPデモ→ゲーム→中間デモ→EDの一巡、セーブして再起動後に続きから、
   assets_override で壁テクスチャを差し替えたプロジェクトが見た目に反映。
4. 素材(音源含む)が全件 CREDITS.md+ディレクトリREADMEに記録されている。
   CC-BY素材を使った場合は帰属表示の実装(タイトル/クレジット画面が
   まだ無いため README と CREDITS への記載で暫定対応)を確認。

## 実装上の注意

- 音量・速度等の「ユーザー設定」とプロジェクトデータを混ぜない。
- セーブへ書く構造体には serde(default) を付け、追加フィールドに強くする。
- ゲーム速度はサイクル長の倍率として一元実装(個々のタイマーを触らない)。
- 差し替え機構のパス解決は project.rs に集約(ランタイム・エディター共用)。
