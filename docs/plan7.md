## データモデル

`src/magic.rs`(新設):

```rust
/// 魔法種別。dandan_spec: 能力値変化(16対象)、MP変更、復活(33/50/100%)、
/// 照明(弱/中/強)など27種。判明していない種別は Unknown として予約し、
/// 実データで判明したら追加する(この文書に追記)。
#[derive(Serialize, Deserialize, Clone, Debug)]
pub enum MagicKind {
    StatChange(StatKind),      // MHP/MMP/攻/防/すばやさ/…/栄養価(HP) の16種
    MpChange,
    Revive { ratio_percent: u8 },   // 33 | 50 | 100
    Light { strength: u8 },         // 1=弱, 2=中, 3=強
}

#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct MagicDef {
    pub id: String,
    pub name: String,          // 参考: 全角6文字
    pub description: String,   // 巻物を「見る」で表示(参考: 20字×2行)
    pub mp_cost: i32,
    pub difficulty: i32,       // 必要な魔法知識
    pub kind: MagicKind,
    pub value: i32,            // 変更値(参考範囲±32767)
    pub duration_cycles: u64,  // 0 = 永続
    pub liquefiable: bool,     // 秘薬化の可否
    pub projectiles: u8,       // 光弾数 0〜2
    pub symbol: String,        // 表示用の1〜2文字(グラフィック化は plan10)
}
```

- キャラの習得魔法: `CharacterState.learned: Vec<String>`(magic id)。
- 習得経路: (a) characters.ron に初期習得を書ける
  (b) 巻物アイテムに `teaches: Option<String>` を追加し、
  「見る」→ 魔法知識 ≥ 難易度なら習得(巻物は消滅)。

## 詠唱の仕様

- 詠唱者選択 → 魔法選択 → 対象選択(味方1人/パーティ全体/正面の敵)。
  対象種別は MagicKind から自動決定(StatChange の負値や光弾付きは敵、
  回復系は味方)。
- 消費: MP ≥ mp_cost が必要。詠唱後 MP -= mp_cost。
- **対魔法力**: 対モンスター魔法は
  `有効率 = clamp(100 − 対魔法力/10, 5, 100)%` で抵抗判定(暫定)。
- **光弾**: projectiles > 0 の攻撃魔法は、発光する球
  (小さな emissive スフィア)が正面方向へ飛ぶ演出を挟んでから着弾処理。
  複数発は少し時差をつける。遮蔽(壁)に当たったら消える。
- **復活**: down(気絶)状態のキャラを HP = MHP×ratio% で復帰。
- **照明**: プレイヤー追随 PointLight の intensity/range を強度に応じて
  duration_cycles の間増加(弱1.5倍/中2.5倍/強4倍、暫定)。切れたら戻す。
- StatChange は plan5 の StatEffect(一時効果)として適用し、
  持続0=永続、装備効果と同じ枠組みで管理する。

## 秘薬(液体化)

- データ画面: 習得済みで `liquefiable` な魔法を選び、空容器(EmptyContainer)
  を1つ消費して「秘薬ビン」を生成(ItemInstance に `potion_of: Option<String>`)。
  生成時 MP を消費する。
- 飲む: 「食べる」操作の液体版。詠唱と同じ効果を自分に適用。ビンは空容器に戻る。
- 投げつける(plan6 の投擲を流用): 着弾マスのモンスターに効果適用
  (攻撃系のみ意味を持つ)。ビンは割れて消滅。

## サンプルデータ

magics.ron に8種以上: 回復(栄養価+)、攻撃(光弾2)、防御強化(持続)、
すばやさ強化、MP回復、復活50%、照明中、秘薬向けの回復(液体化可)。
巻物アイテムを items.ron に追加し、レベルに配置。

1. データモデル+v5ローダー+テスト(ラウンドトリップ、習得判定)

   `light`(照明魔法で明るくなった状態)、`potion`(秘薬生成後のデータ画面)
