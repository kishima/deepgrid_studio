## スコープ

### やること

1. **エディター共通フレーム**: 「左に一覧・右に詳細」の共通レイアウト、
   タブでエディター切替(マップ/キャラ/アイテム/モンスター/魔法/イベント/
   プロジェクト設定)。新規作成・複製・削除・保存(全ファイル一括)。
   Undo/Redo をマップ以外のエディターにも拡張(フォーム単位でよい)。
2. **キャラクターエディター**: プロフィール全項目、成長タイプ、能力値、
   初期習得魔法、見た目モデル選択(assets/models/party から)+
   3Dプレビュー(ポートレートと同じ仕組み)。パーティ編成UI。
3. **アイテムエディター**: 全パラメーター(種別/数値/属性/装備箇所/
   効果リスト/変形条件)。効果リストは追加・削除できる表形式。
4. **モンスターエディター**: 全パラメーター+モデル選択+アニメ割当+
   3Dプレビュー。
5. **魔法エディター**: 種別・数値・液体化・光弾・シンボル文字。
6. **イベントエディター**: トリガー/フラグ条件(AND/OR)/遅延/
   アクション列の編集。アクションは種類に応じたパラメーターフォーム。
   マップエディターのトリガー設置と座標連動(設置→イベント編集へジャンプ)。
7. **プロジェクト設定エディター**: LimitsConfig の全項目+プロジェクト名。
   上限引き下げ時に既存データが超過する場合は警告(project.md)。
8. **マップエディター拡張**:
   - モンスター・アイテムの配置(一覧から選択して置く。
     メモリー使用状況表示: 配置数/上限)
   - イベントフラグの初期状態パネル、スイッチ類のリセット
   - **3Dエディットモード**: プレイ画面と同じ一人称でダンジョン内を
     自由移動(壁抜け可の自由移動モード)しながら、正面のセルに
     パレットのブロックを設置/消去できる。2D⇔3D切替ボタン。
     座標表示・隠し属性表示のトグル。ワープ/しかけ床の作動トグル。
   - 新規レベル作成・削除・並べ替え、連絡通路の接続編集
9. **ゲーム内UIの操作改善**: データ画面のドラッグ&ドロップ
   (plan5 でクリック選択に妥協した部分)、キーバインド設定
   (project.md「UIの方針」。RONに保存)
10. 書ける壁の入力UI(鉛筆アイテム所持時、正面の書ける壁に `W` で入力。
    egui ではなく bevy_ui のテキスト入力が必要 → 簡易実装でよい)

### やらないこと(後続planへ)

- グラフィック差し替え(テクスチャ・ポートレート画像等) → plan10
- デモエディター → plan10
- BGM 選択UI(フロアごと) → plan10(音響と同時)

## エディター構成(画面)

```
+--------------------------------------------------------------------+
| [マップ][キャラ][アイテム][モンスター][魔法][イベント][設定]  [Save All] |
+---------------+----------------------------------------------------+
| 一覧          |  詳細フォーム(選択項目)                            |
| [+新規][複製] |  ・エディターごとの入力欄                          |
| [削除]        |  ・3Dプレビュー(キャラ/モンスター)                 |
| item_001 剣  |  ・バリデーション警告(赤字)                        |
| item_002 盾  |                                                    |
+---------------+----------------------------------------------------+
| ステータスバー: 使用状況(アイテム 12/199 等) | 未保存 * | 警告数    |
+--------------------------------------------------------------------+
```

- すべて egui。ゲーム内UI(bevy_ui)とは分離を維持する。
- 保存は「Save All」でプロジェクト全ファイル(project.ron/levels/各種ron)を
  書き出す。バックアップ(.bak)は plan3 の仕組みを踏襲。
- id はエディターが自動採番("item_001")しつつ手動変更可。参照整合
  (存在しない id 参照)は保存時に警告一覧へ。

## 実装ステップ

1. 共通フレーム(タブ・一覧/詳細・Save All・警告パネル)
2. アイテム → モンスター → 魔法 → キャラ(+パーティ編成)の順に
   フォームを量産(共通フォーム部品: 数値スピナー、能力値グリッド、
   効果リスト表、モデル選択+プレビュー)
3. イベントエディター(アクション列の並べ替え含む)
4. プロジェクト設定+超過警告
5. マップエディター拡張(配置・使用状況・フラグパネル)
6. 3Dエディットモード
7. ゲーム内DnD・キーバインド設定・書ける壁入力
8. 検証: `editor` シーンを各タブ版に拡張
   (`editor-items` `editor-monsters` … で各画面を撮影)

## 受け入れ基準

1. ビルド完走、clippy/test 通過(エディターコアのユニットテスト:
   採番・参照整合・上限警告)。
2. `DEEPGRID_DEBUG_SHOT=editor-*` で各エディター画面が撮影できる。
3. 手動確認: **RONを一切手で書かずに**、新しいアイテム・モンスター・
   魔法・イベント・レベルを作ってプレイモードで動かせる。
   3Dエディットモードで壁を置き2Dに反映される。上限超過の警告が出る。
4. 既存 sample プロジェクトが劣化なく開ける・保存できる
   (保存→再読込で同一。ラウンドトリップテスト拡張)。

## 実装上の注意

- フォームからデータへの反映は必ずエディターコア(コマンド)経由にし、
  Undo/Redo と dirty 管理を一元化する(plan3 の EditorState を一般化)。
- 3Dエディットモードはプレイヤーの移動コード(自由移動モード。plan8 の
  SetMoveMode::Free)を流用する。編集操作はレイキャストではなく
  「正面セル」方式で単純に。
- 数値入力はドラッグ+直接入力の両対応(egui DragValue)。
- 使用状況(メモリー使用状況)は LimitsConfig との対比で全リソース分
  表示する(dandan_spec_mapeditor「その他の処理」)。
