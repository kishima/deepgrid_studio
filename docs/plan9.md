# plan9: コンテンツエディター群 — 「Studio」の完成

## この文書について

第9実装計画書(2026-07-14、plan8 完了後の実状に合わせて詳細化済み)。
仕様源は [dandan_spec_things_editor.md](dandan_spec_things_editor.md)(各項目)、
[dandan_spec_mapeditor.md](dandan_spec_mapeditor.md)(設定メニュー/その他の処理)、
[project.md](project.md)「UIの方針」。開発環境の制約は [plan1.md](plan1.md)。

## ゴール

**RONを一切手で書かずに**ゲームを1本作れる状態にする。全コンテンツの
エディター、プロジェクト設定、マップエディターの拡張(配置・3Dモード)、
そしてゲーム内UIの残った操作課題(DnD・キー設定・壁への書き込み)。

## 現状(plan8 完了時点の実状)

- プロジェクト形式 **v6**。`Project` は dir/name/limits/rules/levels(+paths)/
  characters/party/items/monsters/magics を保持。
  **保存APIは `save_level` のみ**(characters/items/monsters/magics/
  project.ron の書き出しは未実装)。`.bak` 退避は save_level が実装済み。
- エディター: src/editor/ = mod.rs(`EditorState`: レベルごとの Undo/Redo・
  dirty 管理、ストローク合成)+ ui.rs(egui。パレット/フロア/レベル切替・
  Save・下フロア下敷き表示。plan8 の新ブロック文字も対応済み)+
  shot.rs(`DEEPGRID_DEBUG_SHOT=editor` を EguiRenderToImage で撮影)。
- データモデルは**全コンテンツ分が完成済み**: Character(全プロフィール+
  portrait+items+magics)/ItemDef(14種別・効果・装備箇所・teaches)/
  MonsterDef(AI・所持品・アニメ名)/MagicDef(種別23実装)/
  EventDef+TriggerKind+StairsLink+wall_texts/LimitsConfig/RulesConfig(hunger)。
- ランタイム: MoveMode(Normal/Free/Locked)が plan8 で存在(3Dエディット
  モードの自由移動に流用可)。データ画面は bevy_ui でクリック選択+
  アクションボタン方式(DnD未対応)。キーバインドはハードコード。
  書ける壁は**閲覧のみ**(書き込みUIなし)。
- 検証: autotest **42ステップ**、DEBUG_SHOT 17シーン、ユニットテスト41件。

## 設計

### 保存(Save All)

- `project.rs` に `save_project(&Project) -> Result<(), Vec<String>>` を新設:
  project.ron / characters.ron / items.ron / monsters.ron / magics.ron /
  levels/*.ron を**まとめて**書き出す(個別ファイルの部分保存はしない —
  参照整合を保つ単位として全保存が最も安全)。各ファイル .bak 1世代。
- 書き出したプロジェクトの再読込が完全一致することをユニットテストで保証
  (既存のレベル・ラウンドトリップテストを Project 全体へ拡張)。
- エディター起動時は Project を `EditorProject`(編集用クローン)に持ち、
  Save All まではディスクに触れない。

### 共通フレーム

```
+----------------------------------------------------------------------+
| [マップ][キャラ][アイテム][モンスター][魔法][イベント][設定]   [Save All *] |
+---------------+------------------------------------------------------+
| 一覧          |  詳細フォーム(選択中の1件)                          |
| [+新規][複製] |  ・エディターごとの入力欄(下記)                    |
| [削除]        |  ・キャラ/モンスターは3Dプレビュー(ポートレート流用)|
| knight ガルド |                                                      |
| mage  メリナ  |                                                      |
+---------------+------------------------------------------------------+
| 警告: 3件 (クリックで一覧) | アイテム 29/199  モンスター 4/56 …       |
+----------------------------------------------------------------------+
```

- **編集はすべてコマンド経由**: `EditCmd`(SetField/Add/Duplicate/Remove/
  PlaceTile/…)を `EditorState` に流し、Undo/Redo・dirty を全タブで一元化
  (plan3 のマップ用 EditorState を包含する形に一般化)。
  egui の即時モードでは「フォーカスを失った時に確定」を規約とし、
  1文字ごとに Undo 履歴を作らない。
- **id 管理**: 新規は "item_001" 形式で自動採番(重複回避)。id 変更時は
  全参照(party/teaches/carry_items/attack_items/events/配置/魔法習得)を
  **自動で追随更新**し、警告パネルに「N件の参照を更新」と出す。
- **検証警告パネル**(保存時+タブ切替時に再計算):
  存在しない id 参照/上限超過(LimitsConfig 比。引き下げで超過した場合も
  ここに出る)/トリガーブロックと EventDef の座標不一致(plan8 の警告を
  ここへ統合)/装備箇所なしで effects だけあるアイテム等の軽微な矛盾。
  警告があっても保存は**できる**(エラーで止めない。作りかけ を許す)。

### 各エディターのフォーム(dandan_spec 準拠の全項目)

- **キャラ**: プロフィール(名/姓/性別/身長/体重/生年月日/年齢/好き嫌い/
  経歴(複数行))、成長タイプ(5択)、能力値グリッド(15+レベル)、
  総合レベル(読み取り専用表示)、初期アイテム(id複数選択)、初期習得魔法、
  モデル選択(assets/models/party/ 走査のドロップダウン)+portrait パス、
  3Dプレビュー。**パーティ編成**(登録キャラから4人、並び順)もこのタブ。
- **アイテム**: 種別(14択)/数値グリッド(するどさ〜収納数)/重要属性/
  装備箇所(6チェック)/効果リスト(表: 対象・増減・持続。行の追加削除)/
  teaches(魔法ドロップダウン)/モデルパス。
- **モンスター**: 全パラメーター(HP〜逃避HP、耐性4種、行動単位・攻撃頻度は
  「255=しない」の注記表示)、移動タイプ/はしご/幅1通路、サイズ(大は
  「不動のみ対応」の注記)、所持品/攻撃用アイテム(各リスト)、経験値、
  モデル+アニメ名5種(存在しない名前は警告)、3Dプレビュー。
- **魔法**: 種別(実装済みバリアントのドロップダウン+パラメーター)、
  MP/難易度/変更値/持続/光弾数/液体化/シンボル文字/説明。
- **イベント**: 一覧はレベル別にグループ表示。トリガー種別+パラメーター、
  座標(マップタブから「このマスのイベントを編集」でジャンプしてくる)、
  遅延、フラグ条件(行: フラグ番号+On/Off、And/Or)、アクション列
  (種類ドロップダウン+種類ごとのフォーム、上下移動・挿入・削除)。
  wall_texts / stairs_links もこのタブで編集(座標+内容の表)。
- **設定**: プロジェクト名、LimitsConfig 全項目、RulesConfig(hunger の
  有効/数値)。上限を現在使用数より下げた場合は即時に警告パネルへ。

### マップエディター拡張

- パレットに**配置レイヤー**を追加: ブロック/アイテム/モンスター/トリガー。
  アイテム・モンスターは一覧から選択して左クリック配置、右クリック撤去。
  配置数とレベル内種類数は使用状況バーに常時表示(メモリー使用状況)。
- トリガーブロック(K/S/P/T)を置いたら、対応する EventDef の空雛形を
  自動生成してイベントタブへリンク(「イベント未設定」警告付き)。
- **フラグ初期値パネル**(64個のOn/Off。その他の処理「イベントフラグ設定」。
  ※初期値はプロジェクトデータに追加: `project.ron` に
  `initial_flags: Vec<usize>`(Onにする番号)を serde default で追加)。
- 新規レベル: 追加(全壁で生成)/削除(参照する stairs_links・イベントの
  警告)/並べ替え(level番号参照の追随更新)。

### 3Dエディットモード

- マップタブ内の [2D/3D] トグル。3D側はプレイと同じ描画+
  `MoveMode::Free`(plan8)で自由移動(WASD/QE/R/F。落下なし・壁抜けは
  しない — 壁抜けが要る場面は2Dで消す方が速い)。
- 正面セルに対して: 左クリック=選択中パーツを設置 / 右クリック=消去
  (Empty化)。パレットは egui のサイドパネルをそのまま共用。
  設置は TileDirty(plan8)で即時メッシュ反映。
- 座標・向きの常時表示(その他の処理「座標表示」)。隠し属性(隠しワープ)は
  3Dモードでは常に可視化(編集中は見えないと困る)。ワープ・しかけ床は
  **エディット中は発動しない**(その他の処理の作動トグルは「常にOFF」固定で
  開始し、トグルUIだけ用意)。

### ゲーム内UIの仕上げ

- **DnD**: データ画面のスロット間ドラッグ(手⇄装備⇄ポーチ⇄リュック、
  床へドロップで置く)。bevy_ui の Interaction+カーソル位置で自前実装
  (egui は使わない)。クリック選択+ボタン方式は**併存**させる
  (アクセシビリティと autotest の経路として残す)。
- **キーバインド設定**: `user_settings.ron`(リポジトリ直下、.gitignore、
  ユーザー設定 — プロジェクトデータではない)に
  `keybinds: HashMap<GameAction, KeyCode>` を保存。タイトル画面はまだ無い
  ため、設定UIはデータ画面に「設定」タブを足す(ゲーム速度等は plan10 で
  同じ場所に合流)。`desired_command`/アクションキーをテーブル参照に
  書き換える(ユニットテスト: 差し替えたバインドで Command が変わる)。
- **書ける壁への書き込み**: 鉛筆(Pencil/RedPencil/BluePencil)を手に持ち、
  正面が WritableWall のとき `W` キー → 1行入力ボックス(bevy_ui の簡易
  テキスト入力。日本語IMEは不可でよい — 英数かな直接入力の制約を明記)
  → wall_texts を**ランタイム側の LevelState**に書く(プロジェクトデータは
  書き換えない。セーブ対象は plan10)。赤/青鉛筆は文字色。

## 実装上の注意

- **編集コマンド経由の原則**(UIコードから直接データを触らない)。これを
  破ると Undo/Redo と dirty が壊れる。レビュー観点として最重要。
- id リネームの参照追随は「更新対象の列挙」を1関数に集約する(参照箇所が
  plan10 以降も増えるため。関数のユニットテスト必須)。
- egui はエディター専用・bevy_ui はゲーム内、の棲み分けを崩さない
  (壁書き込みの入力ボックスを egui で楽をしない)。
- LimitsConfig/RulesConfig の編集は Project のクローンに対して行い、
  ランタイム側の暫定定数(戦闘式など)には触れない(RulesConfig への
  移住は必要が生じた項目だけ、別コミットで)。
- 3Dエディットモードでもポートレート等のプレイ専用システムは起動しない
  (エディターの App 構成に足すのは描画+移動+TileDirty のみ)。
- `user_settings.ron` は .gitignore に追加(ユーザー設定をリポジトリに
  入れない)。
