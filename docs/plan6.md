## データモデル

`src/monster.rs`(新設):

```rust
#[derive(Serialize, Deserialize, Clone, Copy, PartialEq, Debug)]
pub enum MoveType { Ground, Air, None }

#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct MonsterDef {
    pub id: String,
    pub name: String,           // 表示名(参考: 全角11文字)
    pub profile: String,        // プロフィール(2行程度)
    pub max_hp: i32,
    pub attack: i32,
    pub defense: i32,           // 参考レンジ 0〜32767
    pub agility: i32,           // すばやさ(命中/回避)
    pub attack_freq: u32,       // 攻撃頻度。小さいほど頻繁、255=攻撃しない
    pub anti_magic: i32,
    pub body_temp: i32,         // 体温(衝突ダメージ・温度魔法用。実効果はplan7以降)
    pub resist_air: i32,        // 耐空気力(128以上でその空間を好む)
    pub resist_water: i32,
    pub resist_heat: i32,
    pub resist_poison: i32,
    pub wariness: i32,          // 用心深さ(盗み成功率に影響)
    pub regen_cycles: u64,      // 再生速度: 死後この サイクルで復活。0=復活しない
    pub move_type: MoveType,
    pub can_use_ladder: bool,
    pub fits_narrow: bool,      // 幅1ブロック通路を通れるか
    pub sight: i32,             // 視界 0〜40(0=認識しない)
    pub action_unit: u32,       // 行動単位: 移動間隔サイクル。1=最速、255=移動しない
    pub flee_hp: i32,           // HPがこれ未満で逃走
    pub large: bool,            // 大型(下記「サイズ」)
    pub carry_items: Vec<String>,   // 所持品(最大4目安): ドロップ/盗み対象
    pub attack_items: Vec<String>,  // 攻撃用(最大4目安): 投擲してくる
    pub exp: i32,               // 倒した時の経験値(オリジナルに無い項目。暫定追加)
    pub model: String,          // glbパス
    pub anim: MonsterAnims,     // 下記
}

/// glb内のアニメーション割当(インデックスか名前)。KayKit スケルトンは
/// Idle=40 等(assets/models/README.md 参照)。名前指定を推奨。
pub struct MonsterAnims { pub idle: String, pub walk: String, pub attack: String,
                          pub hit: String, pub death: String }
```

- 配置: レベルファイルに `monsters: [(id, x, y, floor, facing), ...]`。
  バリデーション: レベル内の種類数 ≤ `limits.monster_kinds_per_level`(4)、
  配置数 ≤ `limits.monster_placements_per_level`(48)。
- 実行時状態 `MonsterState`: hp、pos(グリッド+フロア)、facing、
  AIステート、行動ゲージ、死亡時刻(再生用)。

### サイズ(設計判断)

詳細仕様の「大」はスプライト横幅が倍(112×84→224×84ドット)であり、
2Dスプライト前提の概念。3D化した本作では:

- `large: true` は**2×2マス占有**として扱う(DM系の慣例に合わせる)。
- ただし plan6 では大型の移動AI(2×2の通行判定)が複雑なため、
  **大型は「移動しない(action_unit=255)個体のみサンプルに置く」**ところ
  まででよい。動く大型対応は必要になった時点で拡張する。

## モンスターAI(サイクル駆動)

`action_unit` サイクルごとに1回行動判定(clock.rs の on_cycle に載せる):

1. **待機/徘徊**: プレイヤーとの距離 > sight なら、確率50%で
   ランダム隣接マスへ移動(通行可能判定は move_type/はしご/幅1通路/
   液体の好み(耐性128以上の液体は通行可+好む、未満は避ける)に従う)。
2. **追跡**: 距離 ≤ sight かつ遮蔽なし(同フロアで壁による単純視線判定)
   なら、プレイヤーへ近づく1歩(貪欲法でよい。経路探索はしない)。
3. **攻撃**: プレイヤーに隣接し向いているとき、`attack_freq` サイクル
   ごとに攻撃。attack_items があれば距離2〜4で投擲攻撃も行う。
4. **逃走**: hp < flee_hp でプレイヤーから遠ざかる1歩。
5. モンスター同士は同一マスに入らない。プレイヤーのマスにも入らない。

## 戦闘

### プレイヤー側の操作

- **攻撃**(キー `Space` またはアクションアイコン): 正面1マスのモンスターに
  手に持ったアイテムで攻撃(素手も可)。攻撃したキャラは
  パーティ内でローテーション(全員が順に振る。オリジナルの個別操作は
  plan9 のUI改善で検討)。
  ※ `Space` は plan2 でドア開閉に使用済み → **正面がドアならドア、
  モンスターなら攻撃**に多重化する。
- **防ぐ**: 次の被攻撃1回のダメージ半減(アイコン/`B`キー)。
- **精神統一**: 集中力を 5×サイクル回復速度で上昇(アイコン/`C`キー)。
  移動・攻撃で中断。
- **投擲**: データ画面で手に持ったアイテムを `T` で正面方向へ投げる。
  射程 = 遠投力/20 + 投げやすさ/50 マス(暫定)。着弾点のモンスターに
  するどさ×投げやすさ由来のダメージ。アイテムは着弾マスに落ちる。
- **盗み**(`V`キー): 正面のモンスターから
  成功率 = clamp(盗みのうで − 用心深さ, 5, 95)% で carry_items から
  1個奪う。失敗するとモンスターは即座に攻撃してくる。

### 計算式(すべて暫定。変更時はこの文書を更新)

```
命中率(%)      = clamp(50 + (攻すばやさ − 防すばやさ)/10 + 持ちやすさ/10, 10, 95)
基本ダメージ    = max(1, するどさ + 攻撃力/10 − 防御力/20)
集中ボーナス    = 基本ダメージ × (現在集中力 / 集中力最大値) × 0.5
最終ダメージ    = round(基本ダメージ + 集中ボーナス)   // 攻撃後、集中力は0に戻る
モンスター側も同式(するどさ=0、attack を主項とする)
```

- 素手のとき するどさ=0・持ちやすさ=50 とみなす。
- 「防ぐ」中は受けるダメージ半減(1回で解除)。

### 死亡・ドロップ・経験値

- モンスター死亡: death アニメーション → carry_items を足元に散らす →
  `regen_cycles > 0` なら死亡地点でその サイクル後に復活(HP全快)。
- 経験値はパーティ生存者で均等割り。レベルアップ閾値 = level × 100(暫定)。
- **レベルアップと成長タイプ**: 上昇量 = 基礎上昇 × 成長係数(level)。
  - 平均型: 係数常に1.0 / 早期開花: level<20 で1.5、以後0.5 /
    大器晩成: level<20 で0.5、以後1.5 / 天才: 常に1.5 / 才能なし: 常に0.2
  - 基礎上昇(暫定): MHP+4、MMP+2、他の能力+1。HPは全快しない(MHP増分だけ増える)。
- パーティ全滅: 「全滅した…」メッセージ+操作不能(復活はplan7の魔法/plan8のイベント)。
  デバッグ用に `DEEPGRID_REVIVE=1` 起動で即復活できるようにしてよい。

## 実装上の注意

- 上限は LimitsConfig 経由。計算式・暫定数値は定数化+コメント。
- AI・戦闘はすべてサイクル駆動(clock.rs)に載せ、フレームレート非依存にする。
- アニメーション再生は props.rs の仕組みを monster 用に一般化して流用。
- モンスターの Transform 移動もグリッド論理+補間(プレイヤーと同じ思想)。
- 攻撃・被弾でモーションを差し替えたら必ず Idle/Walk に戻す(状態機械)。
- `ClusterConfig::Single` 維持。
