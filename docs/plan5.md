# plan5: アイテムとデータ画面

## この文書について

DeepGrid Studio の第5実装計画書。前提知識は [project.md](project.md)、
[dandan_spec_things_editor.md](dandan_spec_things_editor.md)(Item の節が本plan の
仕様源)、[roadmap.md](roadmap.md)、plan1〜4(実装済み)。
開発環境の制約は [plan1.md](plan1.md)「開発環境の前提」を必ず先に読むこと。
素材追加時は CLAUDE.md「素材の記録ルール」に従う。
オリジナル由来の文字数・数値レンジは**参考値であり強制しない**
(project.md「上限値の扱い」)。

## ゴール

アイテムを「定義 → マップに配置 → 拾う → 持ち物管理 → 装備/食べる」まで
一巡させ、**データ画面**(オリジナルの右クリック画面)を導入する。
あわせて耐性パラメーターを初めて実効化する(液体ダメージ)。

## 現状(plan5 開始時点)

- プロジェクト形式 v2(project.ron + levels/ + characters.ron)。
  ラウンドトリップのユニットテストあり。
- パーティ4人がHUD(bevy_ui + PixelMplus、右ステータス+下メッセージ)に
  表示され、落下ダメージ・集中力回復・サイクルクロック(clock.rs)が動作。
- ポートレートは3Dバスト(portrait.rs、RenderLayers 分離)。
- src/props.rs にスケルトン2体+宝箱+樽のハードコード表示が残っている。
- 検証シーン: `DEEPGRID_DEBUG_SHOT=1|fall|ladder|door|props|editor`。

## スコープ

### やること

1. アイテム定義のデータモデル(14種別+全パラメーター)と `items.ron`
2. レベルファイルへのアイテム配置(プロジェクト形式 v3)
3. 床アイテムの3D表示(KayKitモデル+汎用フォールバック表示)
4. 拾う/置く操作
5. インベントリ(手×2・ポーチ・リュック)と装備(頭首体腕腰足)
6. 装備・食べるによる能力増減効果(持続サイクル含む)
7. 重さと運搬力(超過で移動不可)
8. **データ画面**(装備・持ち物・ステータス/プロフィール・ZZZ)
9. 液体ダメージ(水・火・毒×耐性、サイクル駆動)
10. props.rs のアイテム分(宝箱・樽)をレベルデータ配置へ置き換え

- 鑑定(対鑑定力)、照明(温度による明るさ)、潜望鏡・マップ・コンパス・
  鉛筆の固有機能 → plan8以降(種別・パラメーターだけデータに持つ)
- 変形条件12パターンの実行(エントロピー・温度はフィールドのみ) → plan8以降
- 衝突時効果音 → plan10(音響全般)
- アイテムエディター → plan9
- ドラッグ&ドロップ(plan5はクリック選択→クリック配置。DnD化は plan9)

## データモデル

`src/item.rs`(新設):

```rust
/// 種別(dandan_spec: 14種類)。固有挙動は後続planで実装するものも
/// 種別としては最初から全部定義する。
#[derive(Serialize, Deserialize, Clone, Copy, PartialEq, Debug)]
pub enum ItemKind {
    General, Scroll, EmptyContainer, Liquid, Light, Map, Compass,
    Periscope, Pencil, RedPencil, BluePencil, Accessory, GlowStone, TreasureChest,
}

/// 装備箇所。複数指定可(dandan_spec「装備箇所」)。
#[derive(Serialize, Deserialize, Clone, Copy, PartialEq, Eq, Hash, Debug)]
pub enum EquipSlot { Head, Neck, Body, Arm, Waist, Foot }

/// 能力増減効果(装備中は常時、食べたら適用。持続 0 = 永続)。
#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct StatEffect {
    pub stat: StatKind,        // character.rs の全能力+HP変更(栄養価はitems側)
    pub delta: i32,
    pub duration_cycles: u64,  // 0 = 永続
}

#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct ItemDef {
    pub id: String,            // "sword_iron" 等
    pub name: String,          // 表示名(日本語)
    pub kind: ItemKind,
    pub sharpness: i32,        // するどさ(武器威力)
    pub hardness: i32,         // かたさ(防具性能/食べやすさ)
    pub weight: i32,           // おもさ(×100g)
    pub temperature: i32,      // 温度
    pub nutrition: i32,        // 栄養価(食べた時のHP増減。負=ダメージ)
    pub entropy_max: i32,      // エントロピー最大値(変形は後続plan)
    pub anti_magic: i32,       // 対魔法力
    pub anti_appraisal: i32,   // 対鑑定力
    pub anti_impact: i32,      // 対衝撃力
    pub important: bool,       // 重要アイテム(食べる・投げる不可)
    pub throwability: i32,     // 投げやすさ(実効果は plan6)
    pub grip: i32,             // 持ちやすさ(実効果は plan6 の命中計算)
    pub capacity: u32,         // 収納数(容器のみ >0)
    pub equip_slots: Vec<EquipSlot>,   // 空 = 装備不可
    pub effects: Vec<StatEffect>,      // 装備/食べた時の能力増減
    pub model: String,         // 3D表示用glbパス(空なら汎用表示)
}

/// 持ち物の1スロット。容器の入れ子(容器の中の容器)は plan5 では不可とする。
pub struct ItemInstance {
    pub def_id: String,
    pub entropy: i32,          // 現在値(変形実装まで蓄積のみ)
}
```

インベントリ構造(オリジナルのデータ画面準拠):

- **手**: 左手・右手の2スロット。武器・盾も「持つ」だけなら何でも可。
- **装備**: EquipSlot ごとに1つ(頭・首・体・腕・腰・足)。
  `equip_slots` に含まれる箇所にのみ装備可。
- **ポーチ**: 3スロット。**リュック**: 24スロット(初期値。LimitsConfig に
  `pouch_size` / `backpack_size` を追加して設定可能にする)。
- 総重量 = 手+装備+ポーチ+リュックの全 `weight` 合計(×100g表示)。
  **総重量 > 運搬力(×100g換算)のキャラは移動アクションを拒否**し、
  メッセージ「○○は動けない! 荷物が重すぎる」を出す(パーティ移動なので
  1人でも超過なら移動不可)。

プロジェクト形式 v3:

- `project.ron` に `items: "items.ron"` を追加(version: 3)。
- レベルファイルに配置リストを追加:
  `items: [(id: "sword_iron", x: 5, y: 6, floor: 1), ...]`
  (上限 `limits.item_placements_per_level`)。
- v2 以前も読めること(items 無し=アイテムなし)。

## 3D表示と操作

- 床のアイテムはタイル中央に置く(同一タイル複数個は少しオフセットして重ね、
  4個以上は代表1個+個数表示は plan9 のUI改善まで保留、単純に重ねてよい)。
- モデル: `model` が指定されていればその glb、空なら**汎用表示**
  (小さな回転する八面体+種別色)で必ず何かが見えること。
- KayKit Dungeon Remastered から追加モデルを取得してよい
  (GitHub: KayKit-Game-Assets/KayKit-Dungeon-Remastered-1.0、CC0。
  sword/shield/torch/keyring/coin 等。**CREDITS.md と assets/models/README.md
  への記録を忘れない**こと)。
- **拾う**: 正面1マスか足元にアイテムがあるとき `G` キー(Get)で1個拾い、
  空きスロット(手→ポーチ→リュックの順)に入れる。満杯なら
  「持ちきれない」メッセージ。
- **置く**: データ画面でスロット選択後「置く」ボタン → 足元に置く。
- 拾う/置くはメッセージウインドーに流す(「てつのけんを拾った」等)。

## データ画面

キー `Tab`(および `I`)でメイン画面⇔データ画面を切替(オリジナルの
「3Dウインドー右クリック」の現代化。project.md「UIの方針」)。
プレイは**ポーズしない**(オリジナル同様リアルタイム進行。サイクル処理は
動き続ける)。bevy_ui で実装(スクリーンショットに写る必要があるため)。

```
+------------------------------------------------------------------+
| [P1][P2][P3][P4] ← キャラ切替タブ(ポートレート)                  |
+---------------------------+--------------------------------------+
| 装備(人型レイアウト)      | ポーチ [1][2][3]                     |
|   頭 [ ]                  | リュック [1]...[24](6×4グリッド)     |
|   首 [ ]   手L [ ]        |                                      |
|   体 [ ]   手R [ ]        | 選択中アイテムの詳細:                |
|   腕 [ ]                  |  名前/種別/重さ/するどさ/かたさ...    |
|   腰 [ ]                  |  [装備] [はずす] [食べる] [置く]      |
|   足 [ ]                  |                                      |
+---------------------------+--------------------------------------+
| ステータス: 全能力値+総合レベル | プロフィール | 総重量/運搬力    |
| [ZZZ 休息]                                                        |
+------------------------------------------------------------------+
```

- 操作はクリック選択 → アクションボタン(DnDは plan9)。
- **食べる**: `important` でないアイテムのみ。`hardness > 歯の強さ` なら
  「かたくて食べられない」。成功時 HP += 栄養価(負ならダメージ)、
  `effects` を適用してアイテム消滅。
- **ZZZ(休息)**: ボタンで休息開始 → データ画面を閉じるまで、
  10サイクルごとに全員 HP+1・MP+1(最大まで)。モンスター実装前なので
  「安全な場所」判定は不要(plan6 で敵接近中は休息不可にする)。
- ステータス表示に `overall_level()`(総合レベル)を含める。

## 液体ダメージ(サイクル駆動)

plan2 で見た目だけだった液体を実効化する。パーティが液体タイルにいる間、
サイクルごとに判定(clock.rs の on_cycle 系に載せる):

- **水**: `肺活量 < 100` のキャラに 1 ダメージ/サイクル(暫定閾値)。
- **火**: `耐熱力 < 100` のキャラに 3 ダメージ/サイクル(暫定)。
- **毒**: `耐毒性 < 100` のキャラに 1 ダメージ/サイクル+
  離れた後も 32 サイクル継続(暫定。状態異常の器を作る)。
- 閾値・量は暫定値。コメントに明記し、変更時はこの文書を更新。
- ダメージ時はメッセージ+HPバー反映。HP0 は気絶(plan4 と同じ扱い)。

## 実装ステップ

1. **データモデル**: item.rs、project.rs v3 対応、sample に items.ron
   (8種以上: 剣/盾/兜あたりの装備品、パン等の食料、水の入ったビン、鍵
   (important)、松明、装飾品、宝箱、発光石)+ level00 に配置。
   ラウンドトリップのユニットテスト。
2. **床表示と拾う/置く**: 3D表示、Gキー、props.rs の宝箱・樽を撤去して
   レベル配置に移行(スケルトン2体は plan6 まで残す)。
3. **インベントリと重さ**: スロット構造、総重量、移動拒否。
   重量計算のユニットテスト。
4. **データ画面**: レイアウト実装、装備/はずす/食べる/置く、ZZZ、
   ステータス/プロフィール表示。
5. **効果適用**: 装備中の常時効果と食べた時の効果(持続サイクル、
   0=永続)。効果の適用/解除のユニットテスト(装備→はずすで元に戻る)。
6. **液体ダメージ**: サイクル処理と状態異常(毒)の器。
7. **検証シーン**:
   - `items`: 床にアイテムが数個見える位置で撮影
   - `pickup`: アイテムを拾った後、メッセージが写った状態で撮影
   - `data`: データ画面を開いた状態で撮影(装備・ポーチ・リュックが写る)
   - `liquid`: 水に入って数サイクル後、HPが減った状態で撮影
   - 既存シーンが引き続き通ること

## 実装上の注意

- 上限(ポーチ/リュック容量、配置数)は LimitsConfig 経由。
- ItemDef(定義)と ItemInstance(個体)を混同しない。効果の適用は
  CharacterState 側に「装備由来」「一時効果」を分けて持ち、
  装備解除・持続切れで正しく戻ること(基礎値を直接書き換えない)。
- データ画面中もゲームは進む(サイクル処理を止めない)。
- 液体・毒の暫定数値は必ず定数化+コメントで暫定と明記。
- HUD/データ画面とも bevy_ui + PixelMplus(egui はエディター専用)。
- `ClusterConfig::Single`、ポートレートの RenderLayers 分離は維持。
