## この文書について

第11実装計画書(第1期の最終)。仕様源は [project.md](project.md)

## ゴール

「作る人」と「遊ぶ人」を分離する。タイトル画面からゲームを選んで遊べ、
作ったゲームをランタイム同梱で配布でき、Windows では GPU でまともな
速度で動く状態にして第1期を締める。

## 現状(想定: plan10 完了時点)

- 制作(エディター)→プレイ→セーブまで一巡できる。
- 起動は CLI 引数直指定のみ。タイトル画面なし。
- 実行は docker(lavapipe)経路のみ整備されている。

## スコープ

### やること

1. **タイトル画面**: ロゴ(テキストでよい)+メニュー
   (つづきから / はじめから / ゲームを選ぶ / 設定 / エディター / 終了)。
   オープニングデモがあるプロジェクトは「はじめから」で再生。
   キーバインド設定画面もここから。
2. **プロジェクト選択**: `projects/` ディレクトリ(および `--project`)の
   一覧から選択。プロジェクトに掲載用メタデータ(タイトル・作者・一言)
   を追加(project.ron v+1)。
3. **配布形式**: `--export <dir>` で「ランタイムバイナリ+対象プロジェクト
   +必要アセット(assets/ のうち参照分+assets_override)+README」を
   1ディレクトリに書き出す。エクスポート版は `--play-only`(エディター
   無効・タイトルから単一プロジェクト直行)で起動する設定ファイルを同梱。
   zip化は手動でよい(スクリプト `scripts/export-game.sh` を用意)。
4. **Windows ネイティブ GPU 実行**: project.md 記載の構成を実体化。
   - `.cargo/config.toml` に mingw リンカ設定、
     `scripts/deepgrid-run-win.sh`(mycity-simulator の
     mycity-run-win.sh を雛形に。WSLENV による DEEPGRID_* 橋渡し、
     `~/.cache/deepgrid-target-win/` へのビルド、exe をリポジトリ直下へ
     コピーして起動、AdapterInfo の確認ログ)
   - 相対パスI/O(プロジェクト読込・saves/・スクショ)が UNC cwd でも
     動くことを確認
5. **パフォーマンス**: 40×40×5 フルサイズマップでの docker(lavapipe)
   実測。重い場合の優先手当: 床タイルの矩形マージ、壁の隣接面カリング、
   モンスターのアニメ更新間引き。目標: lavapipe で 20fps 以上、
   Windows GPU で vsync 張り付き(60fps)。
6. **品質仕上げ**:
   - パニック撲滅: ロード失敗・不正データはメッセージ+タイトルへ戻る
   - 起動オプション整理(`--help` 出力)
   - README を「遊ぶ人向け」「作る人向け」「開発者向け」に再構成
   - ライセンス表記画面(CREDITS.md の内容をタイトルの「クレジット」で表示。
     CC-BY 素材の帰属義務をここで満たす)
7. 全 planの検証シーン一括実行スクリプト `scripts/verify-all.sh`
   (全 DEEPGRID_DEBUG_SHOT シーンを順に実行し、生成画像の mtime を検査、
   結果を一覧表示)。CI 相当のローカル完全チェック。

## 実装ステップ

1. タイトル画面+プロジェクト選択+メタデータ
2. play-only モードとエクスポート(export スクリプト+検証:
   エクスポート先だけで起動して遊べること)
3. Windows ネイティブ実行(導入手順は project.md 記載。sudo が要る
   mingw インストールは人間に依頼すること)
4. パフォーマンス計測と手当(計測値をこの文書に追記)
5. 品質仕上げ(パニック撲滅・README・クレジット画面)
6. verify-all.sh と最終総点検

## 受け入れ基準

1. ビルド完走、clippy/test 通過、`scripts/verify-all.sh` 全シーン成功。
2. タイトルから sample を選んで OP→プレイ→セーブ→ロード→ED まで
   マウス/キーボードで一巡できる。
3. `--export` した成果物を別ディレクトリに移しても単体で起動・プレイできる
   (エディターに入れないことも確認)。
4. Windows ネイティブ実行で AdapterInfo が NVIDIA+Vulkan になり、
   体感がdocker経路より明確に軽い。
5. 壊れた project.ron を与えてもパニックせずエラー表示でタイトルに戻る。

## 実装上の注意

- play-only の判定は「エクスポート同梱の設定ファイル」+起動フラグの二重で
  (ユーザーが誤ってエディターを開けない程度でよい。DRM的な厳密さは不要)。
- エクスポートの「参照アセットの抽出」は完全性が重要:
  プロジェクトが参照する model/texture/font/bgm のパスを走査して
  漏れなくコピーする(ユニットテスト対象)。
- Windows 経路の検証は人間の目視が必要な項目を明示して依頼すること。
- 第2期候補は roadmap.md に「第2期」節を作って転記する。
